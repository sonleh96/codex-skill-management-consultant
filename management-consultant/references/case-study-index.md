# Case Study Index

Use this index when the user asks for worked examples, case-study precedents, practice cases, templates, or examples of consulting deliverables. The case-study artifacts live at the repository root under `case-studies/` and are not loaded automatically by the skill.

## Selection Rules

1. Read this index before opening a case-study artifact.
2. Start with the Markdown overview for each case.
3. Open PDFs, XLSX, or HTML artifacts only when the user asks for a full deliverable, model, board deck, memo, or artifact-specific detail.
4. Treat company facts and market data in case studies as static examples. Use web search with citations for current company performance, market data, regulation, valuation, or analyst consensus.

## How To Use Case Studies In Answers

- Use case studies as precedent and structure, not as proof of current facts.
- Reference the artifact path when drawing from a case study.
- If adapting a case-study structure to a new client, preserve the method but replace all company facts, benchmarks, and numbers with client-provided or cited current evidence.
- If the user asks for practice, present the prompt first and reveal the worked answer progressively.
